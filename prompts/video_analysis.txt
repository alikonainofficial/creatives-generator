🔎 DEFINITIONS

Main Subject

The person who drives the narrative and appears most prominently in the video.
	•	If multiple people appear, choose the dominant speaker or the most visually centered person.
	•	If no person appears, set main_subject fields to null where appropriate.

⸻

A-Roll

Shots where the Main Subject is clearly visible and speaking directly to camera or performing actions as the narrative focus.

B-Roll

Product shots, scenery, cutaways, screen recordings, montage clips, or voiceover segments where the Main Subject is not visually dominant.

⸻

🎬 FORMAT CLASSIFICATION RULES

format_name must describe structural style — NOT topic.

It must reflect delivery mechanics and editing structure.

Good examples:
	•	"Single-person direct-to-camera hook with fast jump cuts"
	•	"Street interview Q&A format"
	•	"POV skit with punch-in zoom edits"
	•	"Green screen commentary with background overlay"
	•	"Voiceover over screen recording"
	•	"Podcast clip with kinetic subtitles"
	•	"Educational explainer with animated captions"

Bad examples (DO NOT USE):
	•	"TikTok"
	•	"Vlog"
	•	"Marketing video"
	•	"Entertainment content"

If multiple styles appear, choose the dominant one (≥60% runtime).

⸻

TEXT CLASSIFICATION RULE

Every on-screen text block must include a "text_type" field:

- "subtitle" → Text that directly mirrors spoken dialogue.
- "static_overlay" → Headlines, counters, UI labels, titles, callouts, product names, stickers, meme text, or any non-dialogue visual text.

If the text matches spoken dialogue timing and content, classify it as "subtitle".
If it does not directly mirror speech, classify it as "static_overlay".

👁 FACE VISIBILITY RULES

The best_window must represent the clearest, most stable, unobstructed frontal (or near-frontal) view of the Main Subject's face.

Choose a window where:
	•	Face is not blurred
	•	Face is not occluded
	•	Lighting is sufficient
	•	Camera motion is minimal
	•	Face occupies meaningful frame area

If no such moment exists:
	•	is_clearly_visible = false
	•	best_window = null
	•	Provide reason_if_not_visible

⚠ STRICT RULES
	• TRANSCRIPTION SCOPE RULE:
  - spoken_dialogue MUST include ONLY human narration intended as the video's message:
    (a) the main subject talking to camera (A-roll), OR
    (b) creator voiceover narration.
  - spoken_dialogue MUST EXCLUDE any audio that is playing from within the video itself, including:
    • app/phone/tablet/computer playback audio
    • text-to-speech audio coming from the app demo
    • UI sound effects, notification sounds, button clicks
    • any “example audio” the app is outputting during the demo
  - If the video contains an app demo where audio is heard from the app, treat that as B-roll audio and DO NOT transcribe it.
  - If uncertain whether a line is app-playback vs human narration, set that line to null by omitting it (do not guess).

• All human narration that qualifies under TRANSCRIPTION SCOPE RULE must be transcribed exactly in spoken_dialogue.
	•	Include captions/subtitles in on_screen_text if present.
	•	Timestamps must be chronological.
	•	estimated_a_roll_percentage must be 0–100.
	•	If no spoken dialogue exists, return an empty array.
	•	If no on-screen text exists, return an empty array.
	•	If no main subject is visible, set main_subject fields appropriately and face_visibility.is_clearly_visible = false.
	•	Return JSON only.
